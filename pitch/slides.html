Product · 1 of 3 · Architecture
Product · 2 of 3 · Oracle

The oracle runs inside a TEE.

For a trust-minimized setup, the Oracle code that calls the APIs runs inside a Trusted Execution Environment. We use Acurast — a distributed TEE network powered by old phones.
Acurast Processor cluster · each phone Ⓜ a trusted compute node
🔒 Tamper-proof oracle code trust
Without a TEE, an oracle is only as honest as the server running it. Inside a TEE, the hardware itself attests the code is unmodified — nobody can forge a result.
🛡️ Signing key never leaves the chip hardware
The Oracle's signing key is generated and stored inside the secure enclave. Not even the node operator can extract it.
📡 Acurast — distributed TEE on old phones network
Retired Android phones with secure chips become trusted compute nodes. No data center, no single operator — redundant by default.
📝 Publicly verifiable code hash audit
We publish the hash of the Oracle code running inside the TEE. Anyone can verify what's actually executing matches what we claim.
Product · 3 of 3 · Pricing brain

Classifier looks backward. Agent looks forward.

Layer 1 · Historical baseline
Classifier trained on BTS data

Trained on 5 years of US flight on-time data. Predicts delay probability per route.

Delta JFK→LAX, 9am Tue
→ 19% delay probability
Layer 2 · Today's signal
Live-search agent

History doesn't know about today's storm or war zone. The agent web-searches daily and returns a multiplier.

Storm forecast? → ×1.4 premium
War zone? → disable route
Layer 3 · On-chain write
RouteRepricer cron

Walks every whitelisted route daily. Applies the combined output via governance.

Significant drift → update
Unsafe → disable
Cleared → re-enable
BTS data → Classifier → Live-search agent → RouteRepricer cron → on-chain governance

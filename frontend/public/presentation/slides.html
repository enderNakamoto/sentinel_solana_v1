Oracle & Executor

Oracle Infrastructure & Modular Executor

📡
OracleAggregator Contract
Single source of truth for all flight data. Stores status, ETA, and actual arrival times. Decoupled from the Controller — any oracle can write, any contract can read.
✈
Robust Flight State Machine
8-state deterministic lifecycle validated on-chain. Every transition is enforced — no invalid jumps, no ambiguity. Flights move from Active through to Settled with full auditability.
⚙
Modular Executor Pipeline
4 independent cron jobs (keepers) that toggle flight states: fetch, classify, settle, and extend TTL. Each is swappable and independently deployable.
🛡
TEE-Hardened Oracle (Roadmap)
Future hardening: TEE-attested signer (Switchboard On-Demand SGX) replaces the centralised cron with no on-chain changes — authorized_oracle is already a swap-friendly indirection.

Markets & Governance

Automated Market Creation & Modular Governance

⚙
controller.buy_insurance()
On first purchase for a flight, the controller auto-registers a new FlightPool PDA, initialises the OracleAggregator feed, and locks collateral from the vault_program — all in one transaction via 6 chained CPIs.
📜
governance_program
Modular program that whitelists routes with terms — premium, payout, and delay threshold per airline route. The controller reads these via cross-program invocation and enforces them on every purchase.
💻
React Frontend Integration
Flight search, route selection, premium payment, and purchase confirmation — all wired end-to-end to Solana programs via @solana/kit + Codama-generated typed clients.
🔗
Auto-Sync Deployment
Frontend reads program IDs from deployments/devnet-latest.json on every build. Redeploy regenerates the file — no manual config edits.

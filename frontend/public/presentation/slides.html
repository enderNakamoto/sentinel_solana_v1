Deliverable 01 + 05

Oracle Infrastructure & Modular Executor

📡
OracleAggregator Contract
Single source of truth for all flight data. Stores status, ETA, and actual arrival times. Decoupled from the Controller — any oracle can write, any contract can read.
✈
Robust Flight State Machine
8-state deterministic lifecycle validated on-chain. Every transition is enforced — no invalid jumps, no ambiguity. Flights move from Active through to Settled with full auditability.
⚙
Modular Executor Pipeline
4 independent cron jobs (keepers) that toggle flight states: fetch, classify, settle, and extend TTL. Each is swappable and independently deployable.
🛡
TEE-Hardened Oracle (Roadmap)
Future hardening: TEE-attested signer (Switchboard On-Demand SGX or Acurast mobile-TEE) replaces the centralised cron with no on-chain changes — authorized_oracle is already a swap-friendly indirection.

Four Cron Jobs Run the System

every 5m
SettlementExecutor
Settles classified flights, moves capital, processes withdrawals.
daily

Ecosystem Infrastructure

stellar-wallet-mock

Ethereum has wallet-mock and Synpress for automated dApp testing. Nothing like this existed for Stellar. So we built one.

What it does
Mock Freighter wallet for headless E2E testing of Stellar/Soroban dApps with Playwright. No browser extension needed. No manual wallet interaction. Works with any dApp that uses @stellar/freighter-api or @creit-tech/stellar-wallets-kit — zero dApp code changes required.
Why we built it
We needed automated frontend testing for Sentinel but couldn’t find a way to mock wallet interactions in the Soroban ecosystem. Rather than skip frontend tests, we built the tool — and open-sourced it so other developers can use it too.
Features
✓ Intercepts window.postMessage protocol
✓ Signs real transactions & Soroban auth entries
✓ Pre-seeds localStorage for connected state
✓ Works on testnet, mainnet, standalone
✓ CI/CD ready — no extension needed
✓ Zero runtime deps beyond stellar-sdk

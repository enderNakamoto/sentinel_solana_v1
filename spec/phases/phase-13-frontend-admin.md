## Goal

Wire up the four "operator" pages that sit above the footer on every non-landing
page: `/admin`, `/crons`, `/faucet`, `/contracts`. Together they prove every
governance / oracle / flight-pool admin instruction is reachable from the UI
against the live devnet deployment, give us a manual cron simulator until the
HTTP control surface lands, and put a debug grid in front of every program for
demos. The phase ends with the **12 mock-catalog routes whitelisted on devnet**
through the `/admin` UI itself — that data then backs Phase 14 (underwriter)
and Phase 15 (traveler) without any CLI seeding helper.

## Dependencies

- **Phase 12** — frontend bootstrap (5 routes scaffolded with mock data, Wallet
  Standard connect via framework-kit, three modularity rules locked in).
- **Phase 7** — devnet deployment (program IDs, deployer keypair, mock USDC
  mint, oracle/keeper authorities — addresses captured in
  `deployments/devnet-latest.json` and the README live-deployments table).

No on-chain code lands in this phase. Every action is a frontend call into an
already-deployed Anchor program.

## Decisions Made

- **D1 — Single deployment-constants module.** Hard-coded the devnet
  addresses in `frontend/src/config/devnet.ts` (mirroring
  `deployments/devnet-latest.json`) instead of importing the JSON. Keeps the
  bundle deterministic and avoids dragging the gitignored `deployments/`
  folder into the bundler's resolution path. When the deploy script
  regenerates the artifact, copy values into this file by hand. Trade-off
  accepted: a redeploy needs a one-line code change.
- **D2 — `BottomNav` in `<main>`, not in a footer.** The Phase 12 layout has
  no `<footer>` — main is just `<Topbar/> + children`. Mounted `BottomNav`
  inside `<main>` (with `display: flex; flex-direction: column`) so it sits
  at the bottom of every non-landing page consistently. `usePathname() !==
  '/'` gates visibility.
- **D3 — Known-routes lookup beats `getProgramAccounts`.** The plan suggested
  `getProgramAccounts + memcmp` for the route table; replaced with a
  bounded lookup of the 12 mock-catalog seeds (derived via Codama
  `findRoutePda` then `fetchAllMaybeRouteAccount`). Cheaper, no encoding
  quirks, works against any RPC, and enumerates exactly the routes the rest
  of the UI talks about. Future "show ALL routes" would add a separate
  panel; not needed for Phase 13.
- **D4 — Tri-state via on-chain `U64Update` / `U32Update` enums.** The plan
  said "null for revert, current Option<u64> value for override"; the actual
  on-chain ix takes `Keep | Set | RevertToDefault`. UI tri-state buttons
  map directly to those tagged unions via Codama's `u64Update(...)` /
  `u32Update(...)` factories. `Keep` is a real semantic, not a workaround.
- **D5 — `/contracts` is read-only with curated write routing, not 30
  inline forms.** Hand-rolling a write form for every public ix (~30) would
  duplicate every form already implemented on `/admin` / `/faucet`. Each
  program card shows its full read panel + a `writeNote` describing where
  the writes live, plus shortcut links. Settles the open knob from the
  Pre-work Notes ("every public ix vs curated subset" → curated for write
  side, exhaustive for reads). Trade-off: oracle status mutators
  (`set_estimated_arrival`, etc.) and vault keeper-only writes have no
  inline form yet — those callers are the cron daemon (Phase 8–10) which
  is wired through the executor.
- **D6 — `useSendTx` uses `client.transaction.prepareAndSend`.** Framework-kit
  resolves the `WalletSession` as `authority`, handles blockhash + fee-payer
  + simulation internally, and surfaces wallet-standard signing. Codama
  builders take a `TransactionSigner` placeholder via `createNoopSigner(wallet)` —
  signatures get filled in by the wallet at send time. Honors CLAUDE.md
  W009 (simulate-before-send) without us needing to hand-call
  `simulateTransaction`.
- **D7 — Faucet mints in a single tx with idempotent ATA creation.** Combine
  `getCreateAssociatedTokenIdempotentInstruction` + `mint_to` in one
  transaction so first-time recipients don't need a second click. Idempotent
  variant means re-mints to existing ATAs don't fail.

**Key decisions locked in**

- D1 — devnet addresses hard-coded in `src/config/devnet.ts` (mirroring the
  gitignored `deployments/devnet-latest.json`) for build determinism. Update
  by hand on redeploy.
- D2 — `BottomNav` mounted inside `<main>` (no layout footer exists);
  `usePathname() !== '/'` gates visibility.
- D3 — known-seed lookup of the 12 mock-catalog routes replaces the
  originally planned `getProgramAccounts + memcmp` query — cheaper, no
  encoding quirks, enumerates exactly the routes the rest of the UI talks
  about.
- D4 — UI tri-state buttons map directly to on-chain `U64Update` /
  `U32Update` tagged unions (`Keep | Set | RevertToDefault`) via Codama's
  factories. "Keep" is a real semantic, not a workaround.
- D5 — `/contracts` is read-only with curated write routing. Hand-rolling 30
  duplicate inline forms would just copy what `/admin` and `/faucet` already
  ship; instead each program card surfaces a `writeNote` + page-shortcut links.
- D6 — `useSendTx` uses `client.transaction.prepareAndSend({ authority:
  walletSession })`. Codama builders take a `createNoopSigner(wallet)`
  placeholder; the wallet fills signatures at send time. Honors the
  simulate-before-send guardrail without us calling `simulateTransaction`
  ourselves.
- D7 — Faucet combines `getCreateAssociatedTokenIdempotentInstruction` +
  `mint_to` in a single tx so first-time recipients don't need a second
  click; idempotent variant means re-mints to existing ATAs don't fail.
